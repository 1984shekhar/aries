Blueprint Reflection Package Version 1.0. Bundles wishing to use this package must list the package in the Import-Package header of the bundle's manifest. For example: Import-Package: org.osgi.service.blueprint.reflect;version="[1.0,2.0)" This package provides a reflection based view of the configuration information for a given module context. The top-level mapping between configuration elements and reflection types is as follows: LocalComponentMetadata, ComponentValue
